Самостоятельная работа по теме: «Автоматизация тестирования LMM»

Проект демонстрирует следующие паттерны построения агентских схем:

RAG (Retrieval-Augmented Generation) через Фреймворк LangChain (оптимизированный Retriever).
Observability (Monitoring) через Langfuse (трейсинг цепочек, дебаг, токены).
RAGAS(Retrieval Augmented Generation Automated Scoring) — через  фреймворк для автоматической оценки RAG-систем
Pytest-Через фреймворк для тестирования программного обеспечения на языке Python 

Стек технологий

• LLM_base: Mistral 7B (через Ollama) • Vector DB: Qdrant (через Ollama) • Embeddings: BAAI/bge-m3 (через Ollama) • Observability (Monitoring) через Langfuse • Framework: LangChain, uv
• LLM_ragas: gpt4-o (через Open AI) - для выполнения оценки RAG-системы 


📂 Структура проекта

• rag_qdrant_ragas.py — Точка входа для основного алгоритма.
• test_rag_chain_evaluation.py — Точка входа для функции тестирования .
• ragas_fun.py — точка входа функции запуска оценки RAG-системы.
• Conver_data_for_chank.py — функция разделяющая информацию из входного текстового файла на текстовые смысловые блоки (чанки).
• data/ — Папка с текстовым файлом для базы знаний.

Описание сервиса:

Сервис предназначен для проверки корректности работы RAG сервиса, на основе формирования референтных значений входного вопроса к системе и референтного ответа, которые с промежуточными данными
поступают в LLM_RAGAS для оценки корреткности ответа.  Основные шаги алгоритма: 

1) для формирования текста ответа на обращение клиента. Ответ формируется из данных векторного хранилища. Алгоритм работы следующий :
   
      в векторное хранилище загружены данные в следующей структуре:
              a. context = вопрос из обращения b. metadata =шаблон ответа на вопрос
              структура данных приведена на скрине в папке data/
      
      в векторном хранилище происходит поиск наиболее совпадающих вопросов к реферному вопросу 
      метаданные( answer) по найденным вопросам поступают в LLM_base, на основе которых формируется ответ клиенту.

2)  сформированный ответ LLM_base (response), референтный вопрос (Question), референтный ответ (Answer), найденные документы (связка вопрос-ответ)     
поступают в LLM_ragas, где выполняется оценка корреткности и близости ответов.

3)  основной алгоритм обвернут в функцию тестирования, которая сравнивает результат работы функции оценки RAG-системы с пороговыми значениями и выдает соответствующий результат.  
   
Подключен LangFuse (http://localhost:3000), для мониторинга работы запросов к OLLAMA и OpenAI (скрины приложены в папке /data)
