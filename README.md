Самостоятельная работа по теме: «Автоматизация тестирования LMM»

Проект демонстрирует следующие паттерны построения агентских схем:

RAG (Retrieval-Augmented Generation) через Фреймворк LangChain (оптимизированный Retriever).
Observability (Monitoring) через Langfuse (трейсинг цепочек, дебаг, токены).
Стек технологий

• LLM: Mistral 7B (через Ollama) • Vector DB: Qdrant (через Ollama) • Embeddings: BAAI/bge-m3 (через Ollama) • Observability (Monitoring) через Langfuse • Framework: LangChain, uv

📂 Структура проекта

• OTUS_RAG_Qdran.py — Точка входа.

• Conver_data_for_chank.py — функция разделяющая информацию из входного текстового файла на текстовые смысловые блоки (чанки).

• data/ — Папка с текстовым файлом для базы знаний.

Описание сервиса:

Сервис предназначен для формирования текста ответа на обращение клиента. Ответ формируется из данных векторного хранилища. Алгоритм работы следующий :

в векторное хранилище загружены данные в следующей структуре:
a. context = вопрос из обращения b. metadata =шаблон ответа на вопрос
структура данных приведена на скрине в папке data/

в векторном хранилище происходит поиск наиболее совпадающих вопросов к вопросу заданному клиентом

метаданные по найденным вопросам поступают в LLM, на основе которых формируется ответ клиенту.

Для упрощения интерфейса в алгоритме записан вопрос клиента, на который сервис дает ответ.
